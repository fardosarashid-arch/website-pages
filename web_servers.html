Web_servers

A web server is fundamentally a computer that services HTTP requests — meaning it stores, processes, and delivers web content to clients (usually web browsers) over the internet using the HTTP protocol. This makes the correct choice from the given options: "a computer servicing HTTP requests.

How web servers work.

Least, there are seven stages of how the web server works, from searching to displaying web pages. Here is the explanation:

  • The user enters a request via the browser
  • The browser gets the IP address of the domain and looks it up to the DNS serve
  • The browser requests the required data via HTTP/HTTPS
  • The web server looks for the data requested by the browser on the hosting
  • The browser then connects to the web server and sends the requested web file request via HTTP
  • The web server receives the request and responds by sending the requested web file via the HTTP protocol. However, if the data is not found, the web server will send an error page
  • The browser then displays information to the user

Functions of a Web Server

  • Transfer Data: Sends requested web pages (text, images, videos) to the user's browser. If not found, it returns a 404 error.
  • Cache Management: Clears unused cache to improve website speed and performance.
  • Security Check: Uses HTTP/HTTPS to ensure secure communication between the server and the browser.

Examples of Web Servers

  • Apache: Popular, easy to use, supports PHP and SSL.
  • IIS: Microsoft web server used on Windows, supports multiple protocols like HTTP and FTP.
